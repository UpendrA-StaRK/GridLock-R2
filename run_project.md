# GridLock R2 — Fresh PC Run Guide

> **Use this file when running on a new machine cloned from GitHub.**
> GitHub does NOT contain: raw data, processed parquet files, checkpoints, or outputs.
> Everything must be generated from scratch by running the notebooks in order.

---

## What GitHub Has vs What You Must Generate

| Location | In GitHub? | Action |
|---|---|---|
| `src/`, `configs/`, `notebooks/` | ✅ Yes | Cloned automatically |
| `requirements.txt` | ✅ Yes | Cloned automatically |
| `data/raw/*.csv` | ❌ No (gitignored) | **Copy manually from USB / original PC** |
| `data/processed/*.parquet` | ❌ No (gitignored) | Generated by notebooks 01–03 |
| `data/outputs/` | ❌ No (gitignored) | Generated by notebooks 04–06 |
| `checkpoints/` | ❌ No (gitignored) | Generated by notebook 04 |
| `venv/` | ❌ No (gitignored) | Create fresh (Step 2 below) |

---

## Step 0 — Clone the Repo

```powershell
git clone <your-github-repo-url> GridLock_R2_Transfer
cd GridLock_R2_Transfer
```

---

## Step 1 — Place the Raw Dataset ⚠️ MANUAL STEP

Copy the raw CSV into the project **before running anything else**.
The pipeline will crash on the first notebook without this file.

Target path (create the folder if it does not exist):
```
data\raw\jan to may police violation_anonymized791b166.csv
```

Copy from: your original PC / USB drive / shared drive.
This file is **read-only** — never modify it.

---

## Step 2 — Create the Virtual Environment

Open **PowerShell** in the project root folder and run:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

> If PowerShell blocks script execution:
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
> Then re-run the `Activate.ps1` line.

---

## Step 3 — Install All Dependencies

```powershell
pip install -r requirements.txt
```

If `requirements.txt` is missing anything, use the full explicit line:
```powershell
pip install pandas numpy scikit-learn xgboost lightgbm catboost folium tqdm loguru pyyaml pyarrow matplotlib scipy seaborn shap
```

---

## Step 4 — Create Output Directories

Git does not track empty folders. Create them now:

```powershell
New-Item -ItemType Directory -Force -Path "data\raw", "data\processed", "data\outputs", "checkpoints"
```

---

## Step 5 — Launch Jupyter

```powershell
.\venv\Scripts\jupyter notebook --notebook-dir="notebooks"
```

This opens a browser tab at `http://localhost:8888`.
If the browser does not open automatically, copy the URL (with `?token=...`) from the terminal output and paste it into Chrome.

---

## Step 6 — Run All Notebooks In Order

> **Run every notebook fully (Run All Cells) before moving to the next one.**
> Each notebook saves files that the next one depends on.

| # | Notebook | What it does | Approx time |
|---|---|---|---|
| 1 | `00_run_guide.ipynb` | Audits which files exist, confirms what is missing | ~5 s |
| 2 | `01_eda.ipynb` | Schema validation, null audit, EDA → saves `data/processed/eda_summary.json` | ~1 min |
| 3 | `01b_features.ipynb` | Row-level feature engineering + label encoders → saves `data/processed/features_row_level.parquet` | ~3 min |
| 4 | `02_cluster_tuning.ipynb` | DBSCAN eps/min_samples grid search → saves tuning plots to `data/processed/` | ~5 min |
| 5 | `03_clustering.ipynb` | DBSCAN zones + CIS table + zone×hour grid → saves `zone_hour_grid.parquet`, `cis_table.parquet` | ~8 min |
| 6 | `04_training.ipynb` | Trains 6 models (XGBoost / LightGBM / CatBoost × hour / day) → saves 30 checkpoints | ~10–15 min |
| 7 | `05_inference.ipynb` | Loads winner checkpoint → generates enforcement HTML map + day schedule CSV | ~30 s |
| 8 | `06_shap.ipynb` | SHAP feature importance plots + validation gate report | ~4 min |

**Total cold-start time: ~30–35 minutes**

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'xgboost'` | venv not activated. Run `.\venv\Scripts\Activate.ps1` first |
| `FileNotFoundError: zone_hour_grid.parquet not found` | You skipped notebook 03. Run notebooks in order. |
| `FileNotFoundError: jan to may police violation...csv` | Raw CSV not placed in `data\raw\`. See Step 1. |
| Jupyter URL does not open | Copy the `http://localhost:8888/?token=...` line from the terminal and paste into Chrome |
| `AssertionError: LEAKAGE DETECTED` | Date columns in the CSV have unexpected format. Run notebook 01 first and check validation output. |
| `catboost_info/` folder appears in root | Normal — CatBoost creates this automatically. It is gitignored. |
